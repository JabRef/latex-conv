# CONFLICTS.md

Log of every merge conflict and skipped/non-representable entry encountered while
generating `conversion-table.tsv` from Biber, latex2unicode (tomtung) and JabRef's
`HTMLUnicodeConversionMaps` snapshot. Generated by `tools/GenerateTable.java`; do not
edit by hand — re-run the generator instead.

## tomtung `Unary.combining` vs Biber `diacritics`

Biber is the canonical encode source for accent commands. Bare-letter Biber spellings
(`\"`, `` \` ``, `\'`, `\^`, `\~`, `\=`, breve, `\.`, `\r`, `\H`, `\v`, `\d`, `\c`, `\k`,
`\b`, `\t`) already exist and are kept as the encode-preferred row; 0 identical (command, codepoint) pairs were skipped as duplicates. tomtung's additional
long-form aliases (`\grave`, `\acute`, `\hat`, `\tilde`, `\bar`, `\overline`, `\breve`,
`\dot`, `\ddot`, `\mathring`, `\check`, `\underline`, `\underbar`, `\vec`, `\textcircled`)
were added (0 row(s)) as `alt` decode spellings for the same combining
codepoints.

## tomtung `Unary.subscripts`

maps to a *different* styled char in the two sources is handled by the general

## tomtung `styles` (`\mathbb`/`\textbb` etc.)

Only the `\math*` command spelling is emitted per (family, base char) pair; the `\text*`
alias (same target character) is skipped rather than double-emitted. This is a deliberate
Phase 1 simplification, not a real conflict - full alias round-tripping is out of scope.

## tomtung `Escape.escapes`

Added 396 command rows (category `greek` or `symbols`). Skipped 11 entries that are whitespace/markup rules or non-command tokens
(`~`, `\;`, `\:`, `\,`, `\quad`, `\qquad`, the raw `\\` -> newline rule, and bare
`-`/`--`/`---`), not table-representable. `\DJ` is a special case: tomtung's Scala map
literal has `"\\DJ" -> "Ð"`, which collides with `\DH` and looks like an upstream typo
(it should be Đ, U+0110, matching Biber's own `DJ` entry); Biber's correct `\DJ` -> Đ row
is kept and tomtung's `\DJ` -> Ð target is discarded rather than added as an `alt`.

## tomtung `Binary.frac`

Added 15 `\frac{n}{d}` vulgar-fraction rows (category `symbols`).
Everything else `Binary` computes (general `\frac{n}{d}` parenthesization for pairs not in
this table) is procedural, not table-representable - handled in emitter code later.

## tomtung `UnaryWithOption` (`\sqrt`)

Almost entirely procedural (builds a superscripted radix index for arbitrary `n`); only
the two fixed glyphs U+221B (cube root, `\sqrt[3]{}`) and U+221C (fourth root,
`\sqrt[4]{}`) were extracted as table rows. Plain square root (index blank/`"2"`) is not
re-added since Biber's `surd` -> √ already covers it. Everything else is not
table-representable - handled in emitter code later.

## tomtung `Style.scala`

Pure command aliasing (`\bf` -> `\textbf`, `\cal` -> `\textcal`, `\it` -> `\textit`,
`\tt` -> `\texttt`), not separately table-representable - handled in emitter/parser code
later.

## JabRef `CONVERSION_LIST`

Added 488 rows. Skipped 15 entries with no codepoint (or no usable LaTeX text after normalization) and 28 entries with codepoint <= 128, mirroring
`HTMLUnicodeConversionMaps`'s own static initializer, which only populates its
`*_UNICODE_*` maps for codepoints > 128 - ASCII punctuation is already covered by
Biber's `symbols`/`punctuation` groups. Latex text is normalized by stripping matching
outer `{...}` brace pairs (JabRef-internal grouping noise) while preserving `$...$`
math-mode wrappers and any inner `{arg}` verbatim, e.g. `{\textexclamdown}` ->
`\textexclamdown`, `{{\S}}` -> `\S`, `$\pm$` stays `$\pm$`.

## JabRef `ACCENT_LIST`

Added 75 candidate diacritics rows (same shape as Biber's
`diacritics` group: bare accent trigger token -> combining mark). Most duplicate Biber
exactly and are removed by exact-row dedup; genuine differences (e.g. token `b` is used by
JabRef for both U+0331 "macron below" and U+0332 "underline", while Biber's own `\b` is
U+0331) are resolved by the general cross-latex-spelling conflict pass below.
